## Mobile Considerations

- Bottom navigation (not hamburger menu)
- Cards are full-width, swipeable
- Match percentage is prominent (big colored badge)
- One-tap "Generate Proposal" 
- Pull-to-refresh on tender list
- Push notifications for new matches + deadline reminders
- Offline mode for saved tenders

---

## Key UX Principles

1. **Match score is king** â€” Always show the % match prominently
2. **Show the gap** â€” "You're missing X" is more actionable than just %
3. **One-click to value** â€” Generate proposal should be 1 tap from any tender
4. **Progressive onboarding** â€” Get them to upload docs gradually, not all at once
5. **Deadline anxiety** â€” Use color coding and countdowns
6. **Mobile-first** â€” Contractors are on their phones, not desktops

---

## Tech Stack (Proposed)

- **Frontend:** Next.js 14 + Tailwind + shadcn/ui
- **Mobile:** PWA (installable, works offline)
- **Backend:** Next.js API routes
- **Database:** PostgreSQL (Supabase)
- **Storage:** Supabase Storage (for documents)
- **AI:** OpenAI GPT-4o for proposal generation
- **OCR:** Tesseract or Google Vision (for scanned docs)
- **Auth:** Supabase Auth (email + Google)
